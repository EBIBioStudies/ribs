How to submit data

To submit your functional genomics data to the ArrayExpress collection, go to Annotare and see the submission guide for more information about the submission process and how to access privately held data.

 

How to access private data

First time users: Activate your BioStudies account

At the time of submission, the submitter nominates one email address as the data owner in BioStudies (contact role "submitter") in whose name we have created an account for the BioStudies website and submission tool. Once the experiment has been loaded into the database, this person will receive an email with an activation link. With that account, the data owner can:

  • view and share studies in the database before public release
  • make post-submission updates, such as changing the release date or adding a publication reference
How to activate your BioStudies account

If your data has been loaded into ArrayExpress before the migration to BioStudies, we have created an account for you that you can activate following the steps below.

You must use the same email address that you currently use as ArrayExpress login, which may differ from your Annotare login. If this email address is no longer active, please contact us to update the email address.

Step 1: Identify the email address to use

When your experiment was first loaded, a curator sent an email to the Annotare submitter containing the ArrayExpress access account details. This email account will be your BioStudies account. If you don't have this email any more, don't worry, there are a few more ways to find out which email address to use:

  • The email account you have used to log in to the old ArrayExpress website or access control tool.
  • The email account to which we have sent regular reminders about upcoming releases, with the subject: "E-MTAB-1234: ArrayExpress release in 60/30/7 days time".
  • The email account that is named as "submitter" in the IDF metadata file or the "Contacts" list in the Annotare submission form.

Step 2: Set a password

  • Go to "BioStudies Home > Submit > Submit a Study" to access the BioStudies submission tool
  • Click on Forgot your password
  • Enter your ArrayExpress access account email (from Step 1) and press "Get reset link".
  • Check your emails and follow the activation link to set a password to your account.
  • Log in to view/manage your studies.

Manage your archived data

The following changes can be made to a dataset once it is loaded in the BioStudies database:

  • Release date
  • Publication reference
  • Contact details
These post-submission modifications are done via the BioStudies submission tool. Log in to the tool with your BioStudies account (see above), and you should be able to see your ArrayExpress submissions.

Select the accession to update and click on the pencil icon to edit. Modify the relevant fields in the form and press "Re-submit". The changes will be visible in BioStudies on the following morning.

Get access for reviewers

If you are a submitter, you can find a "secret link" to your experiment on the study page. For this you must be logged in to the BioStudies website (see above). Once you are logged in, search for your accession number and navigate to the study page you want to share. You will find a "Share" icon at the top right.

If you are a reviewer, you should have been provided with a "secret link" that will give you direct access to the private study page. If you do not have the link to access private data connected to a publication, please contact the data submitter, via the journal, to request this information. We cannot provide access to private data without first getting authorisation from the submitter.

 

Programmatic access

Data can be downloaded via FTP or Aspera, following the general instructions for BioStudies data download.

The BioStudies API can be used to programmatically search and access all data in the ArrayExpress collection.

ArrayExpress to BioStudies API migration guide

Due to the migration of ArrayExpress to BioStudies as a collection, there have been the following changes in the API support for ArrayExpress.

XML JSON
XML API responses are being deprecated in favour of JSON. We encourage you to shift to the new JSON schema which should have all the information available in the current AE JSON.
Response bundling
Search API will return a paged response containing only the metadata about the experiments. Results would not be bundled and you'll have to iterate through the set and request each experiment individually.
Endpoint changes
The endpoint has changed from https://www.ebi.ac.uk/arrayexpress/xml/v3 to https://www.ebi.ac.uk/biostudies/api/v1. A list of all searchable fields and their BioStudies equivalent is given below. These fields are in addition to the default BioStudies search criteria fields. Please send all questions and feedback to biostudies@ebi.ac.uk.

POST works too!

In some cases, the request header length is quite large which generates a 400 HTTP response status codes. To avoid this, please use a POST request.

Encode your requests!

As per the HTTP standard, square brackets ([ and ]) are reserved characters. All range requests thus need to be URL encoded to get the correct results. For example, /search?sample_count=[1 TO 5] needs to be encoded as /search?sample_count=%5B1%20TO%205%5D.

 

Learn more about the data in ArrayExpress

MAGE-TAB metadata format

The MicroArray Gene Expression Tabular (MAGE-TAB) format has been developed and adopted by the functional genomics community as a means of representing and communicating data about a functional genomics experiment in a structured and standardised way (Rayner et al., 2006). It was designed for data collection and annotation, as well as for data exchange between tools and databases, including submission tools to public repositories such as ArrayExpress, and has most recently been adapted to accommodate high-throughput sequencing experiments. The full specification outlines the format.

GEO data

Some of the experiments and array designs in ArrayExpress have been imported from the Gene Expression Omnibus (GEO) at NCBI. This regular import has stopped in 2017.

Imported experiments have ArrayExpress accession numbers in the format of E-GEOD-n, where n the same as the number in the original GEO series accession, e.g. GEO accession "GSE12345" would become "E-GEOD-12345" in ArrayExpress. Likewise, for array designs (or platforms), GEO accession "GPL567" would become A-GEOD-567 upon import.

Accession codes

Experiments and array designs in ArrayExpress are given unique accession numbers in the format of

  • E-XXXX-n for experiments
  • A-XXXX-n for array designs

XXXX represents a four letter code and n is a number. E.g. E-MEXP-568, A-UHNC-18.

The four letter code in the accession number indicates the data source. Many of the codes are no longer in active use, i.e. they are not used in accession numbers for new submissions, but remain valid. ExpressionSet R objects

For many ArrayExpress experiments, an "ExpressionSet" object (e.g. E-MTAB-777.eSet.r) is available to download for direct loading into R.

An ExpressionSet is the de facto object defined in the Bioconductor Biobase package for loading and manipulating microarray data in R. These objects are created using the ArrayExpress R package using default parameters. For each experiment, raw (un-normalized) data files are stored in the object as well as other metadata from its MAGE-TAB files, such as study, sample and gene metadata. Please refer to the package ArrayExpress package vignette for more details.
